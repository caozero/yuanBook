# YuanBook

> 基于 C++17 与 SQLite 构建的轻量级跨平台自托管家庭账本，支持多家庭、多账本协作、权限隔离、分类统计、账单导入和可选的 AI 语音记账。

YuanBook 适合部署在家用电脑、迷你主机、NAS 或树莓派上，家庭成员无需安装客户端，通过浏览器即可共同记录和查看收支；账本、账号、会话与配置数据默认保存在本地 SQLite 数据库中，不依赖外部数据库服务。

## 项目状态

当前版本已具备完整的自托管使用闭环，可完成账号初始化、家庭与账本创建、成员协作、收支记录、统计分析、分类维护、账单导入、数据备份和系统管理，处于**可部署使用、持续迭代**阶段。

- **核心功能可用**：手工记账、多账本、统计、分类、成员与权限管理均已落地。
- **跨平台构建可用**：支持 Windows、Linux、Raspberry Pi OS，并提供 Windows 到树莓派的交叉编译脚本。
- **可选 AI 能力**：语音记账和导入分类依赖管理员自行配置兼容接口、API Key、`curl` 与网络环境；未配置时不影响核心记账功能。
- **部署定位明确**：面向个人和可信家庭环境的单机自托管服务，不是托管式云服务，也不建议未经额外安全加固直接暴露到公网。
- **持续完善中**：接口兼容性、自动化测试、安全策略和使用体验仍会继续演进，升级前建议备份实例数据库。

## 功能概览

| 领域 | 当前能力 | 状态与边界 |
|---|---|---|
| 家庭与账本 | 多个账号组成家庭；可以多个家庭,每个家庭可维护多个独立账本，但一般一个账本就足够使用了 | 核心功能 |
| 流水 | 支持收入/支出新增、分页查询、详情展开、编辑、删除，并记录创建者 | 核心功能；修改与删除受成员角色和创建者边界约束 |
| 分类 | 收入、支出分类相互独立，支持一级与二级分类及账本级维护 | 家庭所有者维护；删除在用分类时按领域规则迁移流水 |
| 统计 | 按周、月、年统计收入、支出、结余和分类汇总，并展示与筛选条件一致的流水明细 | 核心功能 |
| 成员协作 | 邀请已有账号、接受/拒绝邀请、查看或取消已发送邀请、移除成员 | 家庭所有者管理；受邀用户必须主动确认 |
| 注册邀请 | 家庭入口可生成加入家庭的限时注册链接；系统后台可生成不自动加入家庭的独立用户注册链接 | 管理功能；两类邀请作用域严格分离 |
| 用户管理 | 管理员可创建、启用、禁用、软删除用户，修改或清空密码；密码变化会注销已有会话 | 系统管理员功能 |
| PID 外部通道 | 用户可创建全局唯一 PID，并将每个 PID 显式绑定到自己有权访问的指定账本，可修改指向或删除 | 可选集成功能；PID 不隐式跟随当前账本 |
| 账单导入 | 上传 `.xlsx` 后由后台任务处理，返回新增、更新、跳过和失败统计，并按来源交易单号去重 | 可选导入功能 |
| 语音与 AI | 将口语文本异步转换为结构化流水，展示解析状态和错误，并允许失败后重新解析；也可辅助导入分类 | 可选能力；依赖兼容接口、API Key、`curl` 和网络 |
| 系统参数 | 管理后台维护服务级参数，并按定义立即应用、软重载或提示重启 | 管理功能 |
| 数据与部署 | SQLite 本地存储、数据库备份重置、工作日志、Windows/Linux/树莓派构建与发布归集 | 部署能力 |

> 当前浏览器客户端以 HTTP 静态页面和 `POST /api/channel` 数据通道为主。源码保留可选 WebSocket 结构，但默认关闭，不作为当前常规部署和使用入口。

## 常规用户使用方式

YuanBook 以单个后台服务配合浏览器页面运行，普通用户不需要安装桌面客户端。

推荐从 GitHub Releases 下载与设备匹配的预编译发布包，解压后直接启动服务，再使用同一局域网内的电脑、平板或手机浏览器访问。
也可架设到云主机,或者局域网透传,这样在外面也可以记账,但暴露在公网,务必注意安全.
为了降低架设复杂度,没有使用https协议,也没有使用加密的数据库,毕竟只是家庭记账而已,如果对数据安全特别在意的请勿使用.
但登录过程使用了数据混淆处理,后续持续使用也是混淆会话,一定程度上增加了一些安全性.

### 可用的安装环境

根据目标设备选择对应的交叉编译或本机构建版本：

- **Windows x64**：适合直接运行在家庭 Windows 电脑、迷你主机或长期在线的家用服务器上。
- **Raspberry Pi OS 64 位（aarch64）**：推荐用于树莓派 4、树莓派 5及其他 64 位 ARM 家庭服务器。
- **Linux x86_64**：适合 Ubuntu、Debian 等常见 Linux 主机、NAS 容器宿主机或家庭服务器。
- **其他 ARM Linux 设备**：可以根据设备架构准备对应的 GNU 交叉工具链和 sysroot 后自行编译。

> 预编译发布包中应同时保留 YuanBook 可执行文件和 `www` 静态资源目录；缺少 `www` 目录时，浏览器管理页面将无法正常加载。

### 启动服务

Windows PowerShell：

```powershell
./YuanBook.exe
```

Linux 或树莓派：

```bash
chmod +x ./YuanBook
./YuanBook
```

服务默认监听 `0.0.0.0:5080`。启动成功后，在运行设备上打开：

```text
http://127.0.0.1:5080/
```

局域网内的其他设备可以将 `127.0.0.1` 替换为服务器的局域网 IP，例如：

```text
http://192.168.1.100:5080/
```

如需自定义监听地址、端口、网页资源目录、数据库位置、日志目录或会话上限，可以使用：

```text
-listen=IP
-http=PORT
-www=PATH
-db=PATH
-log=PATH
-max-sessions=N
```

其中 `-max-sessions=N` 用于限制每个用户最多保留的登录会话数；`N <= 0` 表示不限制。执行 `./YuanBook --help`（Windows 使用 `./YuanBook.exe --help`）可以查看当前版本支持的全部命令行参数。

例如：

```bash
./YuanBook -listen=0.0.0.0 -http=5080 -www=./www -db=./ledger.db -log=./log
```

> 如果需要通过互联网访问，请优先在 YuanBook 前方部署 HTTPS 反向代理，并通过防火墙、访问控制或 VPN 限制入口；不建议直接将服务端口暴露到公网。

### 首次使用

首次部署时，需要先通过命令行创建一个具有 `admin` 权限的系统管理员账号。该命令执行完成后会直接退出，不会同时启动 Web 服务。

Windows：

```powershell
./YuanBook.exe -addUser=admin -pw=请替换为安全密码 -p=admin
```

Linux 或树莓派：

```bash
./YuanBook -addUser=admin -pw=请替换为安全密码 -p=admin
```

其中第一个 `admin` 是示例用户名，可以替换为自定义用户名；`-pw=` 用于设置初始密码，`-p=admin` 表示向该用户授予系统管理权限。命令行中的密码可能被操作系统终端历史记录保存，生产环境应及时清理历史记录或使用受控部署脚本。

省略 `-pw` 或显式传入 `-pw=` 时，新用户密码为空，但登录时不能提交空密码。用户第一次在登录页面输入的非空密码会被系统保存为该账号的正式密码，这一步同时完成首次登录和密码初始化。

已有用户可通过 `-user=用户名 -pw=新密码` 修改密码；使用 `-user=用户名 -pw=` 可清空密码并让用户下次登录时重新设置。两种修改都会注销该用户当前所有登录会话。

1. 使用上述命令创建系统管理员账号并设置初始密码。
2. 正常启动 YuanBook 服务，并在浏览器中打开记账页面。
3. 使用命令行创建时设置的用户名和密码登录。
4. 创建家庭组，并为家庭创建第一个账本。
5. 通过成员邀请或限时注册链接邀请其他家庭成员加入。
6. 家庭成员进入账本后即可记录收入和支出，并在“统计”页面按周、月或年查看收支、结余与分类明细。

## 角色与权限边界

YuanBook 同时存在“系统权限”和“家庭角色”两层授权，二者不能互相替代：

| 身份 | 主要权限 |
|---|---|
| 系统管理员（`admin`） | 访问系统后台，管理用户和服务级参数，创建独立用户注册链接；不会仅因拥有 `admin` 权限而自动成为任意家庭的所有者 |
| 家庭所有者（`owner`） | 管理所属家庭的账本、分类、成员、已有用户邀请和家庭注册链接 |
| 家庭普通成员（`member`） | 访问已加入家庭的账本，记录流水并执行领域规则允许的维护操作 |
| 流水创建者 | 流水会保留创建账号；编辑和删除除了账本访问权外，还会受到创建者及家庭角色规则约束 |

系统管理入口与家庭管理入口彼此独立。家庭注册链接会把新账号加入指定家庭；系统后台生成的独立用户注册链接只创建账号，不自动加入任何家庭或账本。

## 管理与设置功能

### 家庭与账本管理

家庭所有者可以创建用于不同场景的账本；家庭成员可以在自己有权访问的家庭账本之间切换。

每个账本拥有独立的分类和流水数据，适合将家庭日常、旅行、装修等收支分开管理。

### 分类管理

家庭所有者可以添加、编辑和删除收入或支出分类，并通过一级、二级分类建立符合家庭习惯的分类体系。

删除已被流水使用的分类时，系统会按照业务规则处理关联流水，避免产生无效分类引用。

### 成员与邀请

家庭所有者可以邀请已注册用户加入家庭，也可以生成限时注册链接供新成员创建账号。已有用户邀请支持接收方接受或拒绝，发送方可查看并取消尚未完成的邀请。

通过命令行新建且密码为空的用户，第一次登录时需要在登录页面输入一个非空密码；本次输入会被保存为该账号的正式密码，后续登录必须使用同一密码。

通过限时注册链接注册的用户会在注册页面直接设置密码，不需要再执行空密码账号的首次登录初始化流程。

成员邀请需要被邀请人确认；未处理的邀请可以查看或取消，过期邀请会失效。

家庭成员共享家庭下的账本访问范围，但创建账本、维护分类和管理成员等操作仍受角色权限限制。

### 流水、详情与账单导入

用户可以记录支出或收入，并填写金额、日期、分类和备注，也可以修改或删除自己有权限维护的流水。流水列表采用分页加载，记账页和统计页均可展开一致的详情信息；从详情进入编辑时会回到记账表单，并保留当前流水上下文。

账单导入功能支持上传 `.xlsx` 文件。导入任务会在后台处理，页面轮询任务状态并返回识别、成功、新增、更新、跳过和失败等结果。

当导入记录包含稳定的来源交易单号时，重复导入会更新原流水，而不是无条件生成重复数据。

### PID 外部接口与语音记账

用户可以创建个人 PID 外部接口凭证。每个 PID 必须在创建时显式选择一个当前账号可访问的账本；随后可以修改目标账本或删除绑定。外部请求始终定向到 PID 绑定的账本，不会因为浏览器中切换“当前账本”而改变目标。

管理员可以配置语音记账开关、DeepSeek 兼容接口地址、模型名称、API Key、`curl` 可执行文件、超时时间、temperature 和最大 token 数。也可以在启动时通过 `-voice`、`-voice-key`、`-voice-endpoint`、`-voice-model`、`-curl`、`-voice-timeout`、`-voice-temp` 和 `-voice-tokens` 覆盖相关配置。

语音记账采用异步解析：提交口语文本后先生成待解析流水，页面会显示 `pending`、`processing`、完成或失败状态，并在关注期间自动刷新；解析失败且任务不在处理中时，可以手工重新发起解析。

语音记账和 AI 分类属于可选功能；未配置 API Key 或关闭功能时，不影响手工记账、统计和账单导入等核心能力。服务端调用兼容接口依赖可执行的 `curl` 命令，请确保运行账号有权访问对应程序及网络地址。

### 系统管理后台

具有系统管理权限的账号可以访问：

```text
http://服务器地址:5080/admin.html
```

系统管理后台用于维护用户、独立用户注册链接和服务级参数，例如会话、安全策略、日志、账单导入和语音记账配置。用户管理支持创建账号、启用或禁用、修改或清空密码以及保留历史业务数据的软删除；禁用、删除或修改密码会使相关会话失效。

不同参数具有不同的生效方式：部分参数保存后立即生效，部分参数需要业务模块软重载，少数参数需要重新启动服务后生效；后台会根据参数定义显示对应结果。

敏感参数应当只由可信管理员维护，不应通过截图、日志或公开配置文件泄露。

### 数据、备份与重置

默认数据库文件为运行目录下的 `ledger.db`，工作日志默认写入 `log` 目录。可以使用 `-db=PATH` 将数据库放到专门的数据目录，避免与可执行文件和静态资源混在一起。

建议定期备份数据库文件，并在升级、迁移或手工复制数据库前停止服务，确保数据库主文件及可能存在的 `-wal`、`-shm` 文件处于一致状态。

如需清空当前数据库并重新初始化，可以在服务停止后执行：

```powershell
./YuanBook.exe -db=./ledger.db -reset
```

Linux 或树莓派使用：

```bash
./YuanBook -db=./ledger.db -reset
```

重置命令不会启动 Web 服务；它会先把数据库及其 `-wal`、`-shm` 辅助文件复制到数据库同级的 `bak` 目录，确认备份成功后再删除原文件。该操作具有破坏性，执行前必须核对 `-db` 指向的目标路径，并妥善保存备份。

## 开发环境

YuanBook 后端使用 C++17 编写，通过 CMake 管理构建，并使用 cpp-httplib 提供 HTTP 服务、SQLite 保存本地数据。

### 基础依赖

- CMake 3.16 或更高版本。
- 支持 C++17 的编译器。
- Windows：Visual Studio Build Tools / MSVC，或 MinGW-w64。
- Linux / 树莓派：GCC 或 Clang。
- cpp-httplib 单头文件；仓库已包含 `yuanbook/httplib.h`。
- SQLite；Windows 默认可使用仓库内置源码，Linux 可以使用系统 SQLite 或回退到内置源码。

### Windows 本机构建

PowerShell：

```powershell
./build.ps1
```

cmd：

```cmd
build.bat
```

也可以直接使用 CMake：

```powershell
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
```

### Linux / 树莓派本机构建

Debian、Ubuntu 或 Raspberry Pi OS：

```bash
sudo apt update
sudo apt install -y cmake g++ libsqlite3-dev
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
./build/YuanBook -http=5080
```

### Windows 交叉编译树莓派 64 位版本

仓库提供树莓派 aarch64 CMake toolchain 和 PowerShell 构建脚本。准备好 Arm GNU Toolchain 后执行：

```powershell
./build-rpi.ps1 -Arch aarch64
```

树莓派 4 / aarch64 / Release 一键构建也可以使用：

```cmd
build-rpi4-aarch64-release.bat
```

如需指定交叉工具链或目标 sysroot：

```powershell
./build-rpi.ps1 -Arch aarch64 -ToolchainRoot "C:/Program Files (x86)/Arm GNU Toolchain aarch64-none-linux-gnu/14.3 rel1/bin" -Sysroot C:/rpi/sysroot
```

完整的依赖说明、CMake 参数、SQLite 来源选择、交叉工具链配置和发布目录说明，请阅读 [BUILDING.md](BUILDING.md)。


## 安全与部署建议

- YuanBook 面向可信家庭网络和自托管环境设计；公网部署时应使用 HTTPS 反向代理、VPN 或零信任访问网关保护入口。
- 管理员 API Key、数据库文件、日志和备份目录都应限制文件系统访问权限，不应纳入公开仓库或发布到不可信位置。
- 命令行密码和 API Key 可能被终端历史、进程列表或部署日志记录；正式环境优先使用受控脚本，并及时清理敏感记录。
- 修改用户密码或清空密码会注销该用户已有会话；调整最大会话数后，系统会按配置裁剪超额旧会话。
- 当前认证实现用于项目自托管场景；如用于高安全等级或多租户公网业务，应在上线前完成独立安全审计并升级密码存储、传输和密钥管理策略。

## 公开仓库注意事项

将源码推送到 GitHub 前，请确认提交内容中不包含任何真实实例数据或凭证：

- 不要提交 `ledger.db`、`ledger.db-wal`、`ledger.db-shm`、测试数据库、备份数据库、运行日志和本地构建目录。
- 不要提交真实的 AI API Key、用户密码、会话 Token、邀请码、内网地址或包含个人账单的数据文件。
- 发布二进制包时，建议只归集可执行文件、`www`、`content` 和必要说明；除非明确需要演示数据，否则不要附带本地数据库。
- 首次公开前应检查 Git 历史，而不只是当前工作区；敏感信息即使已在后续提交删除，仍可能保留在历史记录中。
- 第三方组件及其许可证应随发布方式一并核对；本项目使用的 cpp-httplib 与 SQLite 相关来源以仓库构建配置为准。
